Source: `scripts/lesson-detail.js`. Renders one lesson from `?id=` (or `?slug=`)
and tracks progress.

## Page wrapper

```html
<section
  data-ms-code="lesson-page"
  ms-code-id-param="id"
  ms-code-slug-param="slug"
  ms-code-table-course="courses"
  ms-code-table-module="modules"
  ms-code-table-lesson="lessons"
  ms-code-table-progress="lesson_progress"
  ms-code-table-breakdown="lesson_breakdown"
  ms-code-table-reference="reference"
  ms-code-table-saved-lessons="saved_lessons"
  ms-code-table-enrollment="enrollments">
</section>
```

## Progress tracking

<Callout>
**A `lesson_progress` row is created the moment the lesson page loads** —
`completed: 0`, `last_watched_at` = now. Marking complete updates that row to
`completed: 1`. This is what makes the course show as *In progress* and the
dashboard continue card resolve. Opening a lesson twice never creates a
duplicate — the script re-checks fresh data before any write.
</Callout>

## Breadcrumbs

`data-ms-code="crumb-learn | crumb-course | crumb-module | crumb-lesson"`, with
`ms-code-detail-page` on each link and `data-ms-field="course.title |
module.title | lesson.title"` for the names.

## Hero + video

| Attribute | Renders |
|---|---|
| `data-ms-field="lesson.title"` | Title |
| `data-ms-field="lesson.description"` | Description |
| `data-ms-code="lesson-meta"` | "LESSON 01 OF 05 · 12 MIN" |
| `data-ms-code="video-player"` | Video — an `iframe`, `video`, or embed `div` |
| `data-ms-code="module-progress-label"` | "3 / 8 LESSONS" |
| `data-ms-code="progress-fill"` | Progress-bar fill |

`video-player` accepts a raw YouTube/Vimeo URL on the lesson — the script
converts it to an embed URL and injects an iframe if needed.

## Completion action

| Attribute | Purpose |
|---|---|
| `data-ms-code="mark-complete"` | The mark-complete / mark-incomplete button |
| `data-ms-show-if="complete"` | Variants `todo` / `done` |
| `data-ms-show-if="enrolled"` | Optional variants `yes` / `no` |

Clicking re-reads fresh progress, then creates or updates the row, and reloads.

<Callout type="warn">
**Mark-complete is gated on enrolment.** If the member doesn't have an active
(non-`dropped`) `enrollments` row for the lesson's course, the Mark complete
button is hidden. Use the optional `data-ms-show-if="enrolled"` variant group
(`yes` / `no`) to design an "Enrol to track progress" CTA that shows in place
of the button when the member isn't enrolled.
</Callout>

## Sidebar curriculum

`data-ms-code="curriculum"` wrapper + `data-ms-code="lesson-template"` rows.
Status variants `data-ms-show-if="status"` with `done · active · locked · todo`.

Each cloned row also receives a `data-status="done|active|locked|todo"`
attribute on its wrapper, so the **whole row** can be styled by attribute,
not only the inner status icon. The row whose lesson page is currently open
additionally gets `data-current="true"` — a **separate** signal from status,
so the lesson you're viewing can be highlighted even when it's already `done`.

```css
[data-status="active"] { /* the resume lesson — purple play icon row */ }
[data-current="true"]  { background-color: rgba(26, 29, 46, 0.06); /* the lesson you're viewing */ }
```

There's also an optional `data-ms-show-if="current"` (`yes` / `no`) variant
group if you'd rather toggle a specific element (e.g. a "● Viewing" badge)
than style the whole row.

<Callout>
**Drip locking is course-wide**, not per-module. A lesson is `active` only if
it matches the **single** first-incomplete position across the whole course
(modules sorted by order → lessons by order). Anything after that position is
`locked`, even if it lives in a later module. So a Module 2 row stays locked
until Module 1's last lesson is marked complete.
</Callout>

## Save + pagination

- Save: `data-ms-code="save-lesson"`, variants `data-ms-show-if="save-lesson"`
  with `saved` / `unsaved`.
- Prev / next: `data-ms-code="prev-lesson"` / `next-lesson` with
  `ms-code-detail-page="/lesson"`; optional `prev-label` / `next-label`. The
  last lesson of a module automatically links forward to the **next module's
  first lesson** (labelled "Next Module →"), and the first lesson links back
  to the previous module's last lesson — so members can keep moving without
  going back to the course page.
- `data-ms-code="view-course"` links back to the course.

<Callout type="warn">
The save-lesson button needs the `saved_lessons` table to exist in Memberstack.
If it's missing the page still loads fine — the save just won't persist.
</Callout>
