title: Course Detail
description: The /course?id=... page — hero, CTA, curriculum, outcomes, instructor, enrolment actions.
---

Source: `scripts/course-detail.js`. Renders one course from `?id=` (or `?slug=`)
and wires every course action.

## Page wrapper

```html
<section
  data-ms-code="course-page"
  ms-code-id-param="id"
  ms-code-slug-param="slug"
  ms-code-table-course="courses"
  ms-code-table-module="modules"
  ms-code-table-lesson="lessons"
  ms-code-table-instructor="instructors"
  ms-code-table-outcome="outcomes"
  ms-code-table-progress="lesson_progress"
  ms-code-table-enrollment="enrollments"
  ms-code-table-saved-courses="saved_courses"
  ms-code-table-saved-lessons="saved_lessons"
  ms-code-table-activity="activity">
</section>
```

The `ms-code-table-*` values are defaults — only include the ones you need to
override.

|---|---|
| `data-ms-field="course.title"` | Title |
| `data-ms-field="course.description"` | Description |
| `data-ms-field="course.category"` | Category |
| `data-ms-field="course.lesson_count"` | Lesson count |
| `data-ms-field="course.hours_estimate"` | Hours estimate |

| `data-ms-code="progress-fill"` | Progress-bar fill |

## CTA + enrolment states

The CTA shows one variant via `data-ms-show-if="cta"`:

| Value | When | Element |
|---|---|---|
| `enroll` | not enrolled | `data-ms-code="enroll-btn"` |
| `reactivate` | paused / dropped | `data-ms-code="reactivate-btn"` |
| `resume` | enrolled & a lesson opened or completed | `data-ms-code="detail-link"` |
| `start` | enrolled, nothing touched yet | `data-ms-code="detail-link"` |

<Callout>
`resume` shows once **any lesson has been opened** — not only after one is
completed. `resume-order` / `resume-cta-label` fill the "Continue · Lesson 03"
text.
</Callout>

Enrolment status variant via `data-ms-show-if="enroll"` with values
`active · paused · dropped · completed · none`, plus label targets like
`data-ms-code="enroll-active-label"`.

Other actions: `data-ms-code="watch-first"`, `save-course`, `drop`,
`hard-unenroll`.

## Curriculum

| Attribute | On | Purpose |
|---|---|---|
| `data-ms-code="curriculum"` | wrapper | Curriculum section |
| `data-ms-code="module-template"` | inner | Module block (cloned per module) |
| `data-ms-code="lesson-template"` | inside module | Lesson row (cloned per lesson) |
| `data-ms-code="module-link"` | optional | Link to the module page |

Lesson rows carry status variants — `data-ms-show-if="status"` with
`done · active · locked · todo`. If no `locked` variant exists in the markup,
locked rows fall back to `todo` and stay clickable.

<Callout>
**Drip locking is course-wide.** Only one lesson is `active` across the whole
course at a time — the single first-incomplete lesson in module-then-lesson
order. Everything after that position is `locked`, even if it lives in a
later module. So Module 2 Lesson 1 stays locked until Module 1 is finished.
</Callout>

| Attribute | Purpose |
|---|---|

## First-lesson preview

An optional section that previews the course's first lesson. Hidden if the
wrapper isn't present.

| Attribute | Renders |
|---|---|
| `data-ms-code="first-lesson"` | Section wrapper |
| `data-ms-field="first-lesson.title"` | First lesson title |
| `data-ms-field="first-lesson.description"` | First lesson description |
| `data-ms-field="first-lesson.duration_minutes"` | Duration |
| `data-ms-field="first-lesson.order"` / `first-module.order` | Lesson / module number |
| `data-ms-code="first-lesson-duration"` | "12 MIN" |
| `data-ms-code="first-lesson-meta"` | "LESSON 01 · MODULE 01" |
| `data-ms-code="preview-thumb"` | Gets a YouTube/Vimeo thumbnail derived from the lesson's `video_url` |
| `data-ms-code="detail-link"` | Link to the first lesson |

## Instructor

`data-ms-code="instructor"` wrapper, with `data-ms-field="instructor.name |
instructor.title | instructor.bio | instructor.avatar"`.

## Confirm modals

`drop` and `hard-unenroll` open confirm modals: `data-ms-code="drop-modal"` /
`reset-modal`, each containing `confirm-yes`, `confirm-cancel`,
`confirm-backdrop`. With no modal element, the script falls back to a native
`confirm()`.

<Callout type="warn">
**Reset** (`hard-unenroll`) deletes the enrolment, all `lesson_progress`, saved
rows, and activity for the course. If an auxiliary table (e.g. `saved_lessons`)
is missing, that part is skipped — the core enrolment + progress wipe still
runs.
</Callout>
